# block
css 에서 \<div>로 박스를 만들게 되면 기본적으로 block으로 인식한다. 이는 `display:block` 이라고 선언한 것과 같다. <br>
이는 화면에서 넓이와 높이가 어떻든 간에 그 박스가 좌우로 공간은 다 차지한다는 것이다. <br>
실제로 width가 200px, background-color: red 이라고 설정해놨을 때 200만큼만 빨간색으로 보이겠지만 사실상 그 width를 초과해서 좌우는 모두 block으로 채워진 것이라서 다음 박스를 삽입하면 자동으로 그 블락 아래에 위치하게 된다. <br>
그런데 여기서 `display: inline-block` 이라고 하면 블락 안에 박스들이 생기기 때문에 두 번째 박스부터 옆으로 나란히 블락 안에 생성된다. <br>

`display:in-line`은 또 얘기가 다르다. 이렇게 선언하면 이는 더이상 박스가 아니라 text로 받아들이기 때문에 width, height 속성이 전부 무시된다. 그래서 글자의 길이만큼만 글자의 크기만큼만 속성이 적용된다. <br>
